# For PMs — How to use Dead Light Framework

## Read these in order

1. **`framework/phases/phase-0.md`** — what Phase 0 (Reckoning) is and when it applies (retrofit vs greenfield).
2. **`framework/pm-calibration-guide.md`** — practical step-by-step at 3 rigor tiers (Tier 1 startup / Tier 2 mid-org / Tier 3 CMMI L4+) for setting your 5 PM Threshold Decisions.
3. **`framework/calibration-standards.md`** — reference catalog of industry standards anchored for each calibration question (skim only; pm-calibration-guide cites this as needed).
4. **`framework/debates/003-phase-0-calibration.md`** — the source debate that anchored the 5 PM commitments to industry standards (read if you want context for why a default is what it is).
5. **`framework/chapters/adeptus-administratum/codex.md`** — the AI-aide Chapter you can invoke to help draft PM commitments + run the Reckoning Team work without becoming a bottleneck (per debate 005).

## Then copy the template

`templates/pm-threshold-decisions-template.md` — copy this into your project, fill the 5 sections, sign off, commit. Without your sign-off, Phase 0 cannot begin (per phase-0 §2 inputs).

## Then either run Phase 0 yourself or invoke Adeptus Administratum

For Tier 1 (startup / solo / small team) projects: invoking an Adeptus Administratum instance (any LLM running under the Codex) can produce the Reckoning Record draft for your review — keeping you from being a bottleneck (per debate 005 motivation).

For Tier 2/3 projects: the Reckoning Team approach with named human members is preferred; Adeptus Administratum supplements the outside-scope perspective per Phase 0 §7.
